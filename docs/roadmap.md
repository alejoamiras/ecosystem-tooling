## Deferred

- **Full run-isolation**: `~/.agents` port registry + `scripts/run/agent.sh` + per-run sandbox `HOME` (only the `NODE_URL`/`L1_RPC_URL` env seam ships today).
- **Upstream tracking automation**: `aztec-nightlies.yml` / `aztec-stable.yml` style workflows that open bump PRs via `scripts/bump-aztec.ts` when new Aztec versions publish.
- **Benchmark numeric regression-blocking**: `--fail-on-regression` mode + calibrated thresholds (gates are non-vacuous report-mode until then).
- **aztec-network org transfer runbook**: scope rename, trusted-publisher re-config, README redirects.
- **`/harden security` whole-repo pass**: scheduled for the org transfer (decision reaffirmed at 5.0.0 approval; a SCOPED `/harden security medium` on fee-payment ran post-5.0.1-release — handover plan Phase 6, report at `audit/security/2026-07-16-shd501/`; 4 findings, 0 Critical/High). Parked for the whole-repo pass explicitly, from the post-impl codex audit (escrow `key_derivation.nr` + sha512 items left with aztec-standards — now the Foundation's surface):
  - **Aztec toolchain supply chain**: `setup-aztec` restores `~/.aztec` from an OS+version-keyed cache and, on miss, runs the `aztec-up` installer with no committed checksum — a persistently-poisoned cache/installer would produce matching rehearsal+release tarballs (payload compare can't catch it). Pin+verify the installer/toolchain digest, or disable the toolchain cache in release builds;
  - **Nargo compile-from-locked-commit**: the release build re-runs `verify-nargo-refs.sh` before compile, which NARROWS but does not close the TOCTOU (aztec compile re-resolves tags independently). True fix: rewrite manifests to the locked commit SHA before compiling, or vendor the deps;
  - Nargo manifest SHA-pinning question (ref-lock enforcement ships now; in-manifest pins deferred);
  - npm-binary deep integrity pinning beyond the exact-version assertion.
- **Structural test debt** (recurring lesson across every phase): the security/release gates are ad-hoc shell + node one-liners first exercised in live choreography (pipefail death, npm `./`-path parsing, TOML-quote escapes, verified-bundle handling all surfaced that way). Add hermetic fixture tests — sample Nargo.toml forms, a captured attestation bundle, mocked `npm view` output — so these gates have adversarial unit coverage instead of comments + operator memory.
- **In-workflow release-conflict preflight**: the runbook re-checks tag/GH-release/version absence immediately before the release dispatch (standards-handover-501 audit D16), but the structural fix is a pre-publish job inside release.yml asserting the `v<version>` tag and GitHub release are absent/identical before `npm publish` runs.
- **Single-source the publishable-package list in release.yml**: `aztec-benchmark aztec-fee-payment` is hand-duplicated across 9 loops + the verify-tarball invocation; the arity assertion guards only the pack step. A workflow-level env (with arity derived from it) makes package add/remove a one-line change (standards-handover-501 code-review finding; deferred to keep the pre-release release.yml diff minimal — pairs naturally with the conflict-preflight item above).
- **Supply-chain report billing gap** (standards-handover-501 audit, fable A5 / codex R1): `bump-aztec.ts`'s report emits publish date/age + provenance-URL presence only — no publisher-continuity column (bunfig.toml's comment overpromises), and upstream `@aztec/*` ships no attestations at all, so the provenance column is vacuous for them. Real controls are structural (OIDC job split, exact pins, Nargo commit-locking). Fix the report's honesty or add post-install lock auditing when upstream starts attesting.
- **Branch protection on `main`**: none exists (verified 2026-07-16 — no protection, no rulesets); PR-gate greenness is procedural only. Add a ruleset (required checks + linear history) at the org-transfer pass.
- **Mock-oracle TXE harness for fee-payment** (cross-cutting item from the 2026-07-16 `/harden security medium`, deferred): `pay_fee`/`mint`/`mint_and_pay_fee` have no active Noir unit coverage (TXE can't seed a FeeJuice nullifier, runs zero gas, and `end_setup()` breaks kernel simulation — the tests are BLOCKED). They're covered only by live-network integration + (for `mint_and_pay_fee`) the assertion-less benchmark. A mock-oracle harness would give the riskiest contract logic a fast unit tier — but it fights the exact TXE limitations that blocked it, so it may need upstream TXE features. (The four SDK/packaging findings F-001..F-004 were IMPLEMENTED in `fee-payment-audit-fixes` (2026-07-17), no release — they ride the next lockstep version bump.)
